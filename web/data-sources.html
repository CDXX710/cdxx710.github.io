GB

geoBoundaries

William & Mary geoLab · est. 2020

Key focus

A standardized, openly licensed database of political administrative boundaries — country, state/province, and district level — for every sovereign state, built specifically to fix the licensing and consistency problems in GADM.

Main differences

Every polygon carries a documented source and license chain back to the original government or humanitarian publisher. Coverage is versioned quarterly, and each boundary is scored for positional and topological quality against a reproducible methodology described in Runfola et al. (2020), PLoS ONE.

Licensing

CC BY 4.0 — commercial use OK

Pros

  • Fully open, attribution-only license — safe for commercial products
  • Machine-readable API and quarterly release cadence
  • Transparent sourcing and per-boundary quality metadata

Cons

  • Fewer administrative levels than GADM in some countries (typically ADM0–ADM2)
  • Younger project — historical archive is shallower
  • Some disputed-territory renderings still evolving release to release

Available attributes

Field Description
shapeName Local or official name of the unit
shapeISO ISO 3166-2 subdivision code, where available
shapeID Unique geoBoundaries identifier, stable across versions
shapeGroup ISO 3166-1 alpha-3 country code
shapeType Administrative level (ADM0–ADM2/3)
GD

GADM

Global Administrative Areas · est. 2009

Key focus

The deepest, most granular administrative hierarchy available for most countries — often down to level 3–5 (municipalities, wards, villages) — assembled from national statistical and mapping agencies.

Main differences

GADM has no restrictions for personal and academic use, but commercial use requires a licensing fee — the single biggest reason teams end up comparing it against geoBoundaries. In exchange, it typically offers more administrative depth and cleaner topology than most free alternatives.

Licensing

Free for academic/non-commercial · fee for commercial

Pros

  • Finest administrative granularity of any global dataset
  • Clean geometries, minimal topology errors
  • Distributed as GeoPackage, Shapefile, KMZ, and R spatial objects

Cons

  • Commercial licensing fee can be a blocker for small teams and products
  • Update cadence is slow and irregular compared to geoBoundaries
  • No public API — bulk downloads only, by country or globally

Available attributes

Field Description
GID_0…GID_5 Hierarchical unique ID at each administrative level
NAME_0…NAME_5 Official name at each level
VARNAME_* Alternate spellings and historical names
TYPE_* / ENGTYPE_* Local designation (e.g. "Provincia") and English equivalent
HASC_* Hierarchical administrative subdivision code
OSM

OpenStreetMap Boundaries

Volunteer-mapped · continuous updates

Key focus

Not a boundary product per se — a crowd-mapped world database from which administrative relations, roads, buildings, land use, and points of interest are extracted. Tools like Geofabrik, the HOT export tool, and osm-boundaries.com turn raw OSM relations into usable boundary layers.

Main differences

OSM-derived boundaries are pulled directly from the live database and can reach much deeper administrative levels — down to neighborhoods and suburbs — in well-mapped regions, whereas GADM offers more consistent coverage but can lag behind fast-changing administrative areas. Quality is only as good as local mapping community activity.

Licensing

ODbL 1.0 — share-alike, attribution required

Pros

  • Continuously updated by a global volunteer community, often within days of real-world change
  • Extends far beyond boundaries: streets, buildings, land use, POIs, transit
  • Free, extractable via API, bulk extracts, or QGIS plugins

Cons

  • Coverage and accuracy vary sharply by region and mapper density
  • ODbL's share-alike clause complicates some closed commercial redistributions
  • No single authoritative "boundaries" file — requires extraction tooling and validation

Available attributes (typical boundary relation)

Field Description

CS

cShapes

Historical state boundaries · since 1946

Key focus

cShapes exists to present historical state boundaries and capitals dating back to 1946, tracking the merger, emergence, disappearance, or dissolution of independent states over time — the tool of choice for political science and conflict research, not current-day mapping.

Main differences

Where GADM and geoBoundaries answer "what does the world look like today," cShapes answers "what did the world look like in any given year." It is coded to both the Correlates of War and Gleditsch–Ward state lists, making it directly compatible with existing conflict and international-relations datasets, and ships an R package for computing inter-capital and inter-border distances.

Licensing

Free for research use, citation requested

Pros

  • Only major dataset built explicitly for time-series, country-year analysis
  • Pre-coded to Correlates of War / Gleditsch-Ward identifiers
  • R package included for distance and adjacency calculations

Cons

  • Country-level (ADM0) only — no subnational boundaries
  • Not maintained on a continuous release cycle like OSM or geoBoundaries
  • Not designed for present-day cartographic or web-mapping use

Available attributes

Field Description
COWCODE Correlates of War numeric state code
GWCODE Gleditsch–Ward numeric state code

CAPNAME / CAPLONG / CAPLAT Capital city name and coordinates
GWSDATE / GWEDATE Start and end date of that territorial configuration
NE

Natural Earth

Public-domain basemap · cartographic focus

Key focus

A public-domain basemap dataset built by volunteer cartographers for making maps that look good at small scales — coastlines, countries, populated places, rivers, and physical features at 1:10m, 1:50m, and 1:110m resolutions.

Main differences

Unlike GADM or geoBoundaries, Natural Earth optimizes for visual generalization rather than administrative precision — boundaries are simplified deliberately so they render cleanly at world or continental zoom levels, not for parcel-level accuracy.

Licensing

Public domain — no attribution required

Pros

  • Truly unrestricted public-domain license
  • Multiple pre-generalized resolutions, ideal for small-scale maps
  • Bundles physical, cultural, and raster layers, not just boundaries

Cons

  • Not suitable where precise, legally accurate boundaries matter
  • Sparse subnational depth compared to GADM or OSM
  • Update cycle is slow and irregular, maintained by volunteers

Available attributes

Field Description
ADMIN / ADM0_A3 Country name and ISO-based 3-letter code
SOVEREIGNT Sovereign parent state, for dependencies/territories
POP_EST Estimated population
CONTINENT / REGION_UN Continental and UN regional grouping
MAPCOLOR7/9/13 Pre-computed color classes for adjacent-country map shading
OM

Overture Maps

Foundation-backed · est. 2022

Key focus

An industry-backed open map dataset (Linux Foundation, with Amazon, Meta, Microsoft, and TomTom among its members) that fuses OSM with commercial and government sources into a single, versioned, cloud-native schema covering places, buildings, transportation, and administrative divisions.

Main differences

Rather than replacing OSM, Overture conflates and cleans it alongside proprietary data, then republishes it as monthly-versioned Parquet releases designed for large-scale data engineering — a very different consumption model from downloading a shapefile.

Licensing

CDLA Permissive 2.0 / ODbL (varies by theme)

Pros

  • Combines crowd-sourced and commercial data for better completeness
  • Cloud-native Parquet/GeoParquet format, built for querying at scale
  • Backed and funded by major mapping and tech companies for long-term stability

Cons

  • Younger, evolving schema — breaking changes between releases are still common
  • Licensing varies by data theme, so it needs checking per layer
  • Heavier tooling requirement (DuckDB/Spark-style workflows) vs. a plain shapefile

Side by side

Pick by use case, not by reputation
The main deciding factor between GADM and geoBoundaries is licensing: GADM is free for academic and non-commercial use, while geoBoundaries is fully open and the safer choice for commercial projects — but GADM often wins on administrative depth where that matters more than licensing.
Source Best for Max depth Update cycle License
geoBoundaries Commercial products needing clean, citable admin boundaries ADM0–ADM2/3 Quarterly CC BY 4.0
GADM Deep subnational analysis, non-commercial or licensed use ADM0–ADM5 Irregular, slow Free (non-comm.) / fee (comm.)
OSM Boundaries Fine-grained, current, locally-verified boundaries ADM0–ADM12* Continuous ODbL 1.0
cShapes Historical / time-series country-level analysis since 1946 ADM0 only Static releases Free, research use
Natural Earth Small-scale, stylized world and continental maps ADM0–ADM1 Irregular Public domain
Overture Maps Large-scale data engineering pipelines, blended sourcing ADM0–ADM4+ Monthly Mixed (CDLA / ODbL)

*Depth in well-mapped urban regions only; varies widely by country and local OSM community activity.
